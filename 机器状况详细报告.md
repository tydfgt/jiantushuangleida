#### 5.2.1 OpenCV 4.10.0 编译详情

**源码获取**

| 仓库 | 来源 | 说明 |
|------|------|------|
| `opencv` | Gitee 镜像 (`gitee.com/mirrors/opencv`) | 国内下载快，branch `4.10.0` |
| `opencv_contrib` | GitHub (`github.com/opencv/opencv_contrib`) | 包含 `cudev` 等 CUDA 依赖模块 |
| 源码分卷备份 | 工作区 `opencv-4.10.0-src.part_aa/ab/ac` | 分卷压缩，便于迁移/重装 |

**CMake 关键配置**

```bash
cmake \
  -D CMAKE_BUILD_TYPE=Release \
  -D CMAKE_INSTALL_PREFIX=/usr/local \
  -D WITH_CUDA=ON \
  -D WITH_CUDNN=ON \
  -D OPENCV_DNN_CUDA=ON \
  -D CUDA_ARCH_BIN=8.7 \          # Orin SM 8.7
  -D CUDA_ARCH_PTX="" \           # 不生成 PTX，省时间
  -D ENABLE_FAST_MATH=ON \
  -D CUDA_FAST_MATH=ON \
  -D WITH_CUBLAS=ON \
  -D WITH_CUFFT=ON \
  -D BUILD_TESTS=OFF \
  -D BUILD_PERF_TESTS=OFF \
  -D BUILD_EXAMPLES=OFF \
  -D BUILD_opencv_apps=OFF \
  -D BUILD_opencv_java=OFF \
  -D BUILD_opencv_python2=OFF \
  -D BUILD_opencv_python3=ON \
  -D OPENCV_EXTRA_MODULES_PATH=.../opencv_contrib-4.10.0/modules \
  -D OPENCV_GENERATE_PKGCONFIG=ON \
  ..
```

**编译参数说明**

| 参数 | 说明 |
|------|------|
| `CUDA_ARCH_BIN=8.7` | Orin Nano 的 SM 架构版本，只编译需要的 target |
| `CUDA_ARCH_PTX=""` | 不生成虚拟架构代码，节省编译时间 |
| `ENABLE_FAST_MATH=ON` | CUDA 快速数学优化（牺牲微小精度换速度） |
| `OPENCV_DNN_CUDA=ON` | DNN 模块启用 CUDA + cuDNN 后端 |
| `OPENCV_EXTRA_MODULES_PATH` | 指向 `opencv_contrib`，提供 `cudev` 等 CUDA 依赖模块 |

**编译与安装**

| 步骤 | 命令 | 备注 |
|------|------|------|
| 配置 | `cmake ... 2>&1 \| tee cmake.log` | 检查 CUDA/cuDNN 检测通过 |
| 编译 | `make -j2 2>&1 \| tee build.log` | ⚠️ 必须 `-j2`，防 8GB 内存 OOM |
| 安装 | `sudo make install && sudo ldconfig` | 安装到 `/usr/local/` |

> 🕐 编译耗时：`-j2` 约 40~60 分钟；`-j1` 约 90~120 分钟。编译时需确保 8GB SSD swap 已启用。

#### 5.3.3 CH340 驱动安装 (Jetson Orin)

> ⚠️ Jetson Orin Nano 内核 (5.15.185-tegra) **不含** `ch341.ko` 驱动模块，需手动编译安装。

**关键步骤**

| 步骤 | 命令 | 说明 |
|------|------|------|
| ① 卸载 brltty | `sudo apt purge brltty` | brltty 会抢占 CH340 串口 |
| ② 克隆驱动 | `git clone https://github.com/WCHSoftGroup/ch341ser_linux` | WCH 官方 Linux 驱动 |
| ③ 编译安装 | `cd ch341ser_linux/driver && make && sudo make install` | 安装到 `/lib/modules/…/usb/serial/` |
| ④ 加载验证 | 插入设备后自动生成 `/dev/ttyCH341USB0` | `depmod -a` 后支持开机自动加载 |

**brltty 问题**

```bash
# brltty 是盲文终端驱动，会抢占 USB 串口
# 症状: lsusb 看到设备但 /dev 无 tty 节点，dmesg 显示 "claimed by ... while 'brltty' sets config"
sudo systemctl stop brltty
sudo systemctl disable brltty
sudo apt purge brltty  # 推荐彻底卸载
```

**驱动编译细节**

```bash
# WCH 官方驱动，比内核自带的 ch341.c 更稳定
git clone https://github.com/WCHSoftGroup/ch341ser_linux.git
cd ch341ser_linux/driver
make -C /lib/modules/$(uname -r)/build M=$PWD modules
sudo make install     # 安装到内核模块目录 + depmod
# 插入 IMU 后自动加载，设备节点: /dev/ttyCH341USB0
```

> ✅ CH340 驱动已编译安装，开机自动加载，brltty 已卸载。
