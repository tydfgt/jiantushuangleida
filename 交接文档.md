## 1. 项目概述

**目标**: 在 NVIDIA Jetson Orin Nano 上搭建一个多传感器 ROS2 Humble 数据采集平台，同时驱动两个 RPLidar C1 激光雷达和一个 10轴 IMU 惯导模块，所有传感器数据通过 ROS2 Topic 发布。

**关键指标**:

| 指标 | 值 |
|------|-----|
| CPU 占用（三传感器同时运行） | ~15% |
| 内存占用 | ~400 MB |
| 雷达扫描频率 | 10 Hz × 2 |
| IMU 发布频率 | ~10 Hz（出厂默认） |
| 总话题数 | 9（含同步话题） |

**硬件连接**:

```
Jetson Orin Nano USB-A 口
├── USB Hub
│   ├── RPLidar C1 #0 (CP210x, /dev/ttyUSB0)
│   ├── RPLidar C1 #1 (CP210x, /dev/ttyUSB1)
│   └── 10轴 IMU     (CH340,  /dev/ttyCH341USB0)
```

## 3. 环境依赖

### 3.1 系统级

| 组件 | 版本/命令 | 说明 |
|------|-----------|------|
| L4T | 36.5.0 | `cat /etc/nv_tegra_release` |
| 内核 | 5.15.185-tegra | `uname -r` |
| ROS2 | Humble | `/opt/ros/humble` |
| colcon | python3-colcon-common-extensions | 构建工具 |
| pyserial | pip3 install | Python 串口库 |
| CH340 驱动 | WCH ch341ser_linux | 见下文 |

### 3.2 必须卸载

```bash
sudo apt purge brltty   # 盲文驱动，会抢占所有 USB 串口
```

### 3.3 CH340 驱动安装

```bash
# 一次性操作（已在本机完成）
git clone https://github.com/WCHSoftGroup/ch341ser_linux.git
cd ch341ser_linux/driver
make
sudo make install   # 安装到 /lib/modules/$(uname -r)/kernel/drivers/usb/serial/
sudo depmod -a
```

安装后设备节点为 `/dev/ttyCH341USB0`（注意不是 `ttyUSB*`）。

### 3.4 udev 规则

```bash
# RPLidar 规则（单雷达通用）
sudo cp sllidar_ros2/scripts/rplidar.rules /etc/udev/rules.d/

# IMU 规则（CH340）
sudo cp wit_ros2_imu/imu_usb.rules /etc/udev/rules.d/

sudo udevadm control --reload-rules
sudo udevadm trigger
```

> 双雷达场景建议用 `/dev/serial/by-id/` 路径或按 ATTRS{serial} 创建独立 symlink。

## 4. 关键设计决策

### 4.1 为什么用 WCH 官方驱动而非内核 ch341.ko？

| | 内核 ch341.c | WCH ch341ser_linux |
|--|-------------|-------------------|
| Jetson 内核自带 | ❌ 未编译 | ✅ 手动安装 |
| 设备节点 | `/dev/ttyUSBx` | `/dev/ttyCH341USB0` |
| 稳定性 | 一般 | 官方维护，更可靠 |
| 与 RPLidar 区分 | 容易混淆 | 名称一目了然 |

### 4.2 双雷达话题设计

两个 RPLidar C1 节点默认都发 `/scan` 话题 → 冲突。

**解决方案**:
- 不同 `name`: `sllidar_node_0` / `sllidar_node_1`
- `remappings`: `/scan` → `/scan_0` / `/scan_1`
- 不同 `frame_id`: `laser_0` / `laser_1`

### 4.3 为什么 sensors.launch.py 放在 wit_ros2_imu 包里？

- `sllidar_ros2` 是第三方包，不应修改
- `wit_ros2_imu` 是我们创建的包，有完全控制权
- 协同 launch 依赖两个包，放在任一个都可以
- `setup.py` 注册后 `ros2 launch wit_ros2_imu sensors.launch.py` 即可发现

### 4.4 RViz 可视化 TF 变换设计

不同传感器的 frame_id 不同（`laser_0` / `laser_1` / `imu_link`），要在 RViz 同一窗口下同时可视化，需要静态 TF 变换将它们关联到同一坐标系。

**解决方案**:
- `show_dual_lidar.launch.py` 中发布两个 `static_transform_publisher`：
  - `laser_0` → `laser_1`：默认偏移 (0,0,0)，可通过参数 `x/y/z/yaw` 自定义雷达安装位置
  - `laser_0` → `imu_link`：默认偏移 (0,0,0)，将 IMU 数据变换到雷达坐标系
- RViz Fixed Frame 设为 `laser_0`，所有数据统一在雷达0坐标系下显示

## 6. 启动流程（人类版）

### 6.1 首次部署

```bash
# 0. 前置条件
sudo apt purge brltty
# CH340 驱动已安装（见 3.3）
# udev 规则已配置（见 3.4）

# 1. 克隆仓库
git clone https://gitee.com/tydfgt/jiantushuangleida.git ~/jiantu
cd ~/jiantu

# 2. 构建
source /opt/ros/humble/setup.bash
colcon build --symlink-install

# 3. 启动
source install/setup.bash
ros2 launch wit_ros2_imu sensors.launch.py
```

### 6.2 日常使用

```bash
# 确认设备在线
ls -la /dev/ttyUSB* /dev/ttyCH34*

# 一命令启动传感器
source ~/jiantu/install/setup.bash
ros2 launch wit_ros2_imu sensors.launch.py

# 一命令启动 RViz 可视化（另一个终端，需桌面环境）
source ~/jiantu/install/setup.bash
ros2 launch wit_ros2_imu show_dual_lidar.launch.py

# 验证数据
ros2 topic list
ros2 topic echo /scan_0 --once
ros2 topic echo /scan_1 --once
ros2 topic echo /imu/data_raw --once
ros2 topic hz /scan_0
```

### 6.3 重新构建

```bash
cd ~/jiantu
source /opt/ros/humble/setup.bash
colcon build --symlink-install
```

### 7.3 IMU 通信协议速查

```
帧格式 (11 字节): 55 XX D0 D1 D2 D3 D4 D5 D6 D7 CK
  55          帧头
  XX          数据类型: 0x51=加速度 0x52=角速度 0x53=姿态角 0x54=磁力计
  D0-D5       数据 (每2字节小端有符号int16)
  CK          校验和 (前10字节累加取低8位)

加速度转换: raw / 32768.0 * 16.0 * 9.8  → m/s²
角速度转换: raw / 32768.0 * 2000 * π/180 → rad/s
角度转换:   raw / 32768.0 * 180          → °
```

### 7.4 改动须知

1. **不要改 `sllidar_ros2/`** — 这是第三方包，升级时直接覆盖
2. **改 IMU 驱动的统一入口** — `wit_ros2_imu/wit_ros2_imu/wit_ros2_imu.py`
3. **加新传感器** — 在 `wit_ros2_imu/launch/sensors.launch.py` 加新 Node
4. **改话题名** — 修改 launch 文件的 `remappings`
5. **串口权限** — 确保用户在 `dialout` 组，或通过 udev 设置 MODE="0777"

## 8. 调试排查速查表

| 症状 | 第一检查 | 第二检查 |
|------|---------|---------|
| 雷达不响应 | `lsusb` 看 CP210x 在不在 | `sudo modprobe cp210x` |
| IMU 无设备 | `lsusb` 看 CH340 在不在 | `lsmod \| grep ch341` |
| 设备出现又消失 | `ps aux \| grep brltty` | `sudo apt purge brltty` |
| IMU 报端口占用 | `sudo fuser -v /dev/ttyCH341USB0` | `sudo fuser -k /dev/ttyCH341USB0` |
| 雷达 topic 无数据 | `ros2 topic hz /scan_0` | 看 dmesg 有无超时 |
| 构建失败 | `source /opt/ros/humble/setup.bash` | 检查 CMakeLists/setup.py |
| launch 报 `channel_type` | 确认传给 sllidar_node 的 parameters 有 `channel_type` | 值应为 `'serial'` |

## 9. 已知限制与改进方向

- [ ] IMU 节点崩溃后不会自动重启（加 `respawn=True` 即可）

- [ ] CH340 驱动需在每台新 Jetson 上手动编译
